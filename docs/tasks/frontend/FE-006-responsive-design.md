# FE-006: レスポンシブデザイン対応

## 概要
全ページのレスポンシブデザイン対応を完成させ、モバイル・タブレット・デスクトップで快適に利用できるようにする。

## 優先度
**High** - モバイルユーザーは非常に多い

## 難易度
**Medium**

## 前提条件
- [x] FE-001: プロジェクトセットアップが完了
- [x] FE-002: 共通コンポーネントが完成
- [x] FE-003: スレッド一覧ページが実装済み
- [x] FE-004: スレッド詳細ページが実装済み
- [x] FE-005: スレッド作成ページが実装済み

## 実装内容

### 1. ブレークポイント定義
- [ ] ブレークポイントの標準化
  ```typescript
  export const breakpoints = {
    mobile: '320px',
    tablet: '768px',
    desktop: '1024px',
    wide: '1440px',
  };
  ```
- [ ] Tailwind設定への反映（Tailwind使用の場合）
- [ ] CSS-in-JSへの反映（使用の場合）

### 2. レイアウトコンポーネントの対応
- [ ] `Header` コンポーネント
  - モバイル: ハンバーガーメニュー
  - タブレット: 簡略ナビゲーション
  - デスクトップ: フルナビゲーション
- [ ] `Sidebar` コンポーネント
  - モバイル: ドロワー（スライドイン）
  - タブレット: 固定サイドバー（狭い幅）
  - デスクトップ: 固定サイドバー（標準幅）
- [ ] `Footer` コンポーネント
  - モバイル: 縦積み
  - タブレット/デスクトップ: 横並び

### 3. スレッド一覧ページ
- [ ] カード表示の調整
  - モバイル: 1カラム、縦長カード
  - タブレット: 1-2カラム
  - デスクトップ: 2-3カラムまたはリスト表示
- [ ] フィルタ/ソートUI
  - モバイル: モーダルまたはアコーディオン
  - タブレット/デスクトップ: サイドバー
- [ ] ページネーション
  - モバイル: 簡略版（前/次のみ）
  - デスクトップ: フル版（ページ番号表示）

### 4. スレッド詳細ページ
- [ ] 投稿リストの表示
  - モバイル: 縦積み、コンパクト表示
  - タブレット/デスクトップ: 標準表示
- [ ] 投稿フォーム
  - モバイル: フルスクリーンモーダル（オプション）
  - デスクトップ: インライン表示
- [ ] リアクションボタン
  - モバイル: タップしやすいサイズ（44x44px以上）
  - デスクトップ: 標準サイズ

### 5. スレッド作成ページ
- [ ] フォームレイアウト
  - モバイル: 縦積み、フルフォーム
  - デスクトップ: 2カラムレイアウト（ラベル+入力）
- [ ] タグ選択UI
  - モバイル: モーダルまたはドロップダウン
  - デスクトップ: チェックボックスグリッド
- [ ] 送信ボタン
  - モバイル: 固定ボトムバー（オプション）
  - デスクトップ: フォーム内

### 6. タッチ最適化
- [ ] タップターゲットサイズ（最小44x44px）
- [ ] スワイプジェスチャー対応（オプション）
- [ ] ホバー効果の調整（モバイルでは無効化）
- [ ] フォーカス状態の視覚化

### 7. タイポグラフィ調整
- [ ] フォントサイズのレスポンシブ対応
  ```css
  /* 例 */
  h1 { font-size: clamp(1.5rem, 5vw, 3rem); }
  ```
- [ ] 行間・余白の調整
- [ ] 可読性の確保

### 8. 画像・メディア対応
- [ ] レスポンシブ画像（srcset/sizes）
- [ ] アスペクト比の維持
- [ ] 遅延読み込み（LazyLoad）
- [ ] プレースホルダー表示

### 9. パフォーマンス最適化
- [ ] モバイル向けバンドルサイズの削減
- [ ] 重要なCSSの inline化（Critical CSS）
- [ ] フォント読み込みの最適化
- [ ] 画像の最適化（WebP等）

### 10. テスト
- [ ] 各ブレークポイントでの表示確認
- [ ] 実機テスト（iPhone, Android, iPad等）
- [ ] タッチ操作のテスト
- [ ] ランドスケープ/ポートレート切り替えテスト
- [ ] Lighthouse モバイルスコア90以上

## 受け入れ基準
- [ ] すべてのページがモバイルで正常に表示される
- [ ] タブレットで正常に表示される
- [ ] デスクトップで正常に表示される
- [ ] タップターゲットが適切なサイズ（44x44px以上）
- [ ] 横スクロールが発生しない
- [ ] フォントサイズが読みやすい
- [ ] Lighthouse モバイルスコア90以上
- [ ] iOS Safari, Android Chrome で動作確認済み
